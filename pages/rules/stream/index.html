Streaming Guidelines

Guidelines for streamers and instructions on how to set up OBS

The below guide will cover the basics of OBS, and how to get your stream setup! If you have any questions, don't hesitate to ask in #board-chat.

Warning
Please do not show the matches' room ID on the stream! Cover it or change the scene in OBS (to black or an intermission screen, for example) if you're entering it on stream.

OBS Setup

Setting Up Resolution, Frame Rate and Codecs

First, we want to make sure your OBS settings are configured correctly. You can set your resolution, frame rate and codecs in the Video and Output sections of the settings menu.

Common resolutions used include 1920x1080 (1080p), 1280x720 (720p) and 2560x1440 (1440p), you would usually set your resolution to match your monitor's native resolution.

Base (canvas) resolution and output (scaled) resolution should usually be the same.

Your FPS (frame rate) is also set on this page. Most modern systems should be able to handle 60FPS, so set it to this. This matches the framerate of the game, be sure to check your capture card settings aswell, make sure it is outputting at 60FPS too.

It's usually a good idea to switch from software encoding (the default) to hardware encoding in the Output section of OBS settings. This offloads the encoding work to your GPU and can resolve performance issues, especially on weaker devices.

You can also set the bitrate in this menu aswell. See YouTube's bitrate guidelines for details.

Use a high enough bitrate for a clear picture. At 1080p60 (H.264), 12 Mbps is recommended. If you are streaming at 1440p or 4K, you can use higher bitrate if your upload speed allows it.

Run an internet speedtest to check your upload speed for streaming.

Generating Your Stream Overlay

Before opening OBS, generate a personalised stream overlay from the UMKL Discord server using this app command:

/generate-stream-overlay

You can run this command for yourself or for someone else in the UMKL Discord server. It generates an stream overlay image, that you can then download and use in OBS as a source. It displays your Discord username to identify which team's perspective is being shown.

Adding Sources in OBS

Sources are the building blocks of what viewers see on stream. Add them in the order below - sources higher in the list appear on top of those beneath them.

  1. Discord Voice Overlay - Set this up at streamkit.discord.com/overlay and add it as a Browser source in OBS.
  2. Score Overlay - Add as a Browser source using one of the following URLs, depending on your team's position in the match title:
    • Use live.umkl.co.uk if your team is listed first
    • Use liverev.umkl.co.uk if your team is listed second
    Team icons at the top of the screen update automatically when a match starts.
  3. Stream Overlay - Add the URL generated with /generate-stream-overlay as a Browser source.
  4. Video Capture Device - Add your capture card as a Video Capture Device source. This captures the game footage itself.
Setting Up Audio
There are two ways to capture Discord audio:
  • Desktop Audio - captures all system sound, but can cause double audio if Discord audio is also coming through your headset.
  • Application Audio Capture (Beta) - captures Discord directly and avoids double audio. This is the recommended option.
Mute any audio sources that aren't actively in use.

To hear game audio without a splitter: Right-click Video Capture Device in the Audio Mixer, select Advanced Audio Properties, and set Audio Monitoring to Monitor and Output.

Arranging Your Stream Layout

Once your sources are added, arrange them into a clean layout so viewers can follow the match easily.

  • Resize and position each source using the red bounding box handles in the OBS preview window.
  • Keep the score overlay and stream overlay fully visible and unobstructed - don't let the video capture source cover them.
  • Leave space near the room ID area of your gameplay so it can easily be covered or cut to an intermission scene, per the warning above.
  • Consider creating a separate Intermission scene (e.g. a static image or colour) to switch to between races or if the room ID needs to be shown.

Entering The Stream Key

To connect OBS to your team's YouTube channel:

  1. Open Settings in OBS and go to the Stream tab.
  2. Set the service to YouTube - RTMPS.
  3. Select Use Stream Key (advanced).
  4. Paste in your team's stream key.
Don't have a stream key?
Contact a board member to receive your team's stream key.

Multiple Profiles (Optional)

OBS profiles let you save separate stream configurations. If you stream to both a personal account and the UMKL channel, create a profile for each and switch between them using the Profile dropdown menu.

Audio Mixing

Before going live, check that your audio levels are well balanced. Use the image below as a guide. As a general rule:

  • Game audio should be audible but not dominating
  • Discord voice should be clear and sitting above background noise
  • Mute any sources not in active use

Testing Your Stream Before a Match

Always do a test run before a match starts, rather than discovering an issue once it's live. A few minutes of testing beforehand can save a lot of trouble during the match itself.

  1. Check OBS's Stats window - Open it via View → Docks → Stats. Watch for Dropped Frames and high Render/Encoding Lag, both of which usually mean your bitrate, resolution or frame rate are too high for your hardware or upload speed.
  2. Test on your personal YouTube account first - You can switch to a personal OBS profile (see Multiple Profiles above) and go live privately or unlisted there for a few minutes before the match. This lets you confirm video, audio and overlays are all working without anyone else seeing it, and without risking an accidental public test on the UMKL channel.
  3. Check the delay - YouTube live has a short delay before your stream appears. Confirm the video is smooth and not stuttering once it catches up.
  4. Check your audio - Make sure Discord voice and game audio are both audible, balanced, and in sync with the video, and that no source is unintentionally muted or left unmuted.
  5. Check your overlays - Confirm the score overlay is showing the right match and updates once it starts, and that the stream overlay is displaying the correct username.
  6. Double check the room ID is hidden - Switch through your scenes to make sure the match's room ID isn't visible anywhere on stream, per the warning at the top of this page.
  7. End the test stream before the match begins, then switch back to your team's OBS profile and stream key, and go live on the team channel once you're ready for the match.

Additional Notes

  • Stream overlay username: Your overlay will show your Discord username to identify your team's perspective. Make sure your Discord DM settings are configured as you'd like before going live.
