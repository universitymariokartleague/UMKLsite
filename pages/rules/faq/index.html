Frequently asked questions concerning the Discord server and Cheep Cheep app

How Cheep Cheep Works

@Cheep Cheep is the custom Discord bot that runs the UMKL. It manages everything from team creation and player sign-ups, to running matches, verifying results and tracking stats, all through Discord slash commands.

In short, Cheep Cheep:

  • Creates and manages team channels, roles and categories
  • Handles player sign-ups and substitutions for matches
  • Verifies match results and calculates points, SP and standings
  • Powers live score overlays and stream graphics
  • Feeds the matches, teams and leaderboard data shown across this website via its API

See the Discord/Cheep Cheep Commands Documentation page for a full list of commands. The rest of this page answers common questions about how everything fits together.

Why can't I see my team's channel? What should I do?

Members of your team need to use the /join command to join, this will give them the correct role and allow them to view their team's channel.

Each team's channels (announcements, chat and voice channel) are only visible to members with that team's Discord role - everyone else, including other teams, can't see them at all. Picking your team with /join is what gives you this role and makes the channels appear.

I joined the wrong team! What should I do?

Please contact a board member and they will place you in the right team.

  • Their Nintendo Switch friend code (optional)

How are match results verified once they're submitted?

After a match, a board members will submit both teams' scores using /add-match-results. This posts a verification embed in the board's results channel, which will then be approved or rejected, penalties can also be added if applicable.

Once approved, Cheep Cheep automatically generates the results and winner graphics, updates the team standings, distributes SP to players, closes out the match's Discord event, and posts an announcement to the results channel:

What do the leaderboard and profile commands show?

/leaderboards shows 8 different leaderboards: Season and Lifetime Points, and Season and Lifetime Best Match Score, each for both Teams and Individuals, plus an SP leaderboard.

/user-profile shows an individual's team, wins, matches played, podiums, highest finish, career points and SP. /team-profile shows a team's location, institution, season position, win/loss record, points, penalties and championships.

How are matchups decided each round?

At the start of a new round (/start-next-round), teams are paired up based on the previous round's results - teams with a similar number of points tend to be paired together. The board then sets the actual matchups with /add-next-round-pairings, and match organisation threads are created automatically for each pairing.

A Discord account is required for @Cheep Cheep to function properly, so please ask all members of your team to join the Discord server.

What does Cheep Cheep actually run on?

Cheep Cheep is a Python Discord bot built with discord.py, using slash commands, modals, buttons and pagination for all of its interactions.

Team, player, match and season data is stored in a SQLite database, alongside a handful of JSON files for things like live scores, SP history and season pairings. All of the graphics you see - results images, winner announcements, team standings, stream overlays and YouTube thumbnails - are generated on the fly using Pillow.

This website talks to that same database through a small API (api.umkl.co.uk), which is what powers the matches, teams and leaderboard data shown across the site.

What the hell is a "hex"? Like… the "Lost Hex" from Sonic Lost World?


No. Shut up. Stop it. Well you know how our decimal system has 10 possible numbers (0-9), this is called a base-10 numbering system. Hexadecimal (hex) is a base-16 numbering system, meaning its possible values are from 0-9 and A-F (A-F essentially representing 10-15).

Software developers and system designers widely use hexadecimal numbers because they provide a convenient representation of binary-coded values. Each hexadecimal digit represents four bits (binary digits), also known as a nibble. For example, an 8-bit byte is two hexadecimal digits and its value can be written as 00 to FF in hexadecimal. A hex code is split into three parts, Let's say we have the hex colour code #1BAA8B. Ignore the # and split the string into three parts of two.

We get 1B, AA and 8B. Since these are hexadecimal values, this means that two hexadecimal numbers cover 16² values, so 0-255 as denary. If we convert each group into denary, we get 27, 170 and 139.

Each of these numbers we got represent the RGB values of a colour - red, green and blue. AKA RGB(27, 170, 139). This is another colour format also used, so essentially, a hex colour code is a compact version of RGB.
